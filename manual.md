## Установка Codex
1. Войдите в свой аккаунт с активной подпиской ChatGPT.
2. Перейдите на `https://chatgpt.com/codex`.
3. В блоке «Начать работу с Codex» выберите вариант установки:
   - расширение для IDE (VS Code, Cursor, Windsurf);
   - терминал (CLI) — можно использовать в любом терминале, в любой IDE.
   Расширение удобнее для большинства задач, так как имеет полноценный UI; CLI подходит, если нужен минимальный интерфейс и быстрые команды из терминала.
4. Следуйте инструкциям установки для выбранного варианта (все очень просто).

## Практичные сценарии использования
1)  Помощь в анализе проблемы (почему «не так», можно запускать параллельно с работой над другой задачей, тем самым экономя свое время)
- Пример: «найди, что отвечает за скролл на странице Х, почему он появляется и предложи точечный фикс». Сodex найдет компонент, цепочку взаимосвязей, суть проблемы.

2) Разбор падений сборки и типовых ошибок
- Пример: «разверни ошибку до причины, укажи точный файл и минимальный фикс», «почему здесь `never/unknown` и как типизировать».

3) Точные UI-правки с контекстом дизайна (с этим нужно быть аккуратнее, так как агент не совершен, возможно больше времени понадобится на рефакторинг и доведение к консистентности проекту, чем на правки с нуля)
- Сценарий: подправить файлы стилей, проверить скриншот, сопоставить с Figma. (требует настройки mcp)

4) Масс-правки по файлам
- Пример: обновить импорт/тип/хук во всех контейнерах

5) Документация.
- Codex отлично составляет документацию

6) Доступ к актуальной документации библиотек (требует настройки mcp)
- Можно быстро получить справку по API/примеру без ручного поиска по сайтам.
- Может сверить ваш код с документацией и подсказать, соответствует ли он текущим рекомендациям.

7) Помощь с Git в сложных ситуациях
- Если вы «накосячили», Codex подскажет безопасный путь отката, восстановления или переноса изменений и предложит команды, о которых легко забыть.

8) Code review и риск-анализ
- Codex может провести первичный аудит вашего кода перед отправкой на ревью (экономит время и ваше и ревьюера).

## Рекомендации по использованию
1) Если задача не маленькая, создайте (или попросите Codex создать) `OOP-****.md` с описанием и требованиями (просто скопируйте их), в процессе работы над задачей codex будет актуализировать этот файл. Так агент будет постоянно опираться на файл и не потеряет контекст.
2) Указывать в промте цель и критерий готовности (лучше всего, если это тоже будет добавлено в `OOP-****.md`)   
3)  Давать четкие границы (если это возможно), иначе codex будет дольше искать нужное.
4) Перед выполнением какого-либо действия (особенно если это что-то сложное) уточнять у codex как именно он планирует выполнять задачу
5) Если задача очень сложная и многосоставная, описать ее агенту, попросить добавить план в файл `OOP-****.md` и раздробить на конкретные шаги, идти уже по конкретным шагам в плане (у codex большой контекст, но при выполнении многосоставных задач могут возникать трудности)
6)  Если необходимо просить перечислить риски/регрессии перед выполнением.

## MCP
MCP (Model Context Protocol) — это способ подключать к Codex внешние источники контекста (документацию, базы знаний, внутренние справочники). Ниже — полезные MCP‑инструменты и кратко, как их подключать.

### Context7:
    быстрый доступ к актуальной документации библиотек по запросу.
#### Как добавить и использовать:
1. Получи ключ Context7
* Открой `https://context7.com/dashboard`
* **API Keys -> Create API Key**
* Скопируй ключ.

2. Попроси Codex сам добавить Context7 в конфиг
* Вставь ключ **в промпт ниже** вместо `PASTE_API_KEY_HERE`
* Отправь промпт Codex.

---

#### Промпт для Codex (скопируй целиком и отправь)

```text
Нужно подключить Context7 MCP.

1) Открой файл конфигурации Codex:
- macOS/Linux: ~/.codex/config.toml
- Windows: %USERPROFILE%\.codex\config.toml

2) Если файла нет - создай его.

3) Внутри файла добавь:
[mcp_servers.context7]
command = "npx"
args = ["-y", "@upstash/context7-mcp", "--api-key", "PASTE_API_KEY_HERE"]

4) Сохрани файл.
```

3. Перезапусти IDE или открой новый терминал (если пользуешься CLI), чтобы он перечитал конфиг.
4. Проверка: в Codex попроси обратится к context7
5. Чтобы использовать просто пиши в запросе: "используй context7 для ..."

### Figma MCP

    Доступ Codex к макетам/компонентам/переменным Figma, чтобы генерировать код по дизайну и вытягивать контекст.

#### Как добавить и использовать

1. Установи **Figma Desktop** и открой любой файл. 
2. Переключись в **Dev Mode** 
3. В **Inspect panel → MCP server** нажми **Enable desktop MCP server** и убедись, что сервер запущен.
4. Дальше нужно просто добавить URL сервера в конфиг Codex.

---

#### Промпт для Codex (скопируй целиком и отправь)

```text
Нужно подключить Figma Desktop MCP к Codex.

1) Открой файл конфигурации Codex:
- macOS/Linux: ~/.codex/config.toml
- Windows: %USERPROFILE%\.codex\config.toml

2) Если файла нет — создай его.

3) Добавь в конец:

[mcp_servers.figma_desktop]
url = "http://127.0.0.1:3845/mcp"

4) Сохрани файл.
```

5. Перезапусти IDE или открой новый терминал (если пользуешься CLI), чтобы он перечитал конфиг. 
6. Проверка: в Codex попроси обратится к Figma MCP
7. Чтобы использовать, выбери в Figma Desktop, с включенным Dev-mode, компонент / группу, которую хочешь превратить в код (или забрать отступы и цвета, например), и пиши запрос, этого достаточно, чтобы codex "увидел" с чем нужно работать, либо же дай ссылку на компонент / группу / блок, при выделнии в Dev-mode справа во вкладке inspect есть прям промт с ссылкой, лучше брать только ссылку, так как промт говорит имплементировать сожержимое в код, а это не обязательно то, что хочешь от codex

---

### Browser MCP (BrowserMCP/mcp)

    Управление реальным Chrome для чтения сайтов: открыть страницу, кликать, скроллить, вводить текст, собирать данные, делать скриншоты.

#### Как добавить и использовать

1. Установи расширение Browser MCP в Chrome из Chrome Web Store.

2. Убедись, что установлен Node.js

* MCP-сервер запускается через `npx`, поэтому нужен Node.js.

3. Попроси Codex добавить Browser MCP в конфиг отправь промпт ниже.

---

#### Промпт для Codex (скопируй целиком и отправь)

```text
Нужно подключить Browser MCP (BrowserMCP/mcp) в Codex.

1) Открой файл конфигурации Codex:
- macOS/Linux: ~/.codex/config.toml
- Windows: %USERPROFILE%\.codex\config.toml

2) Если файла нет — создай его.

3) Добавь в конец (или обнови, если уже есть):

[mcp_servers.browsermcp]
command = "npx"
args = ["-y", "@browsermcp/mcp@latest"]

4) Сохрани файл.
```

5. Перезапусти IDE или открой новый терминал (если пользуешься CLI), чтобы он перечитал конфиг.

6) Проверка:
- Открой Chrome
- Открой любую НОВУЮ вкладку
- Нажми на иконку Browser MCP расширения и выбери "Connect" — это закрепит вкладку за MCP
- попроси например: "Перейди на https://google.com и сделай скриншот страницы".
---

### Jira MCP

  Дает Codex доступ к Jira/Confluence/Compass Cloud

#### Как добавить

1. Должен быть установлен **node.js 18+**

2. Попроси Codex добавить Atlassian MCP в конфиг: отправь промпт ниже.

---

#### Промпт для Codex (скопируй целиком и отправь)

```text
Нужно подключить Jira MCP через официальный Atlassian Rovo/Remote MCP Server (через mcp-remote).

1) Открой файл конфигурации Codex:
- macOS/Linux: ~/.codex/config.toml
- Windows: %USERPROFILE%\.codex\config.toml

2) Если файла нет — создай его.

3) Добавь:

[mcp_servers.atlassian]
command = "npx"
args = ["-y", "mcp-remote", "https://mcp.atlassian.com/v1/mcp"]

4) Сохрани файл.
```

5. Перезапусти IDE или открой новый терминал (если пользуешься CLI), чтобы он перечитал конфиг.

6. Проверка:
- Попроси: "Через Atlassian MCP найди все Jira issues, назначенные на меня за последние 7 дней".
- Если откроется браузер — пройди OAuth-логин Atlassian и вернись в IDE.
7. Примеры использования:
   * **Сводка по тикету**: цель, критерии, статус, блокеры, последние комменты, ссылки.
   * **Бэклог**: узнать какие задачи на тебе
   * **Поиск контекста**: найти похожие тикеты, связанные задачи, зависимости.
